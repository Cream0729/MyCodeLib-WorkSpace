2. 简单的Java程序

最初的，以HelloWorld程序分析为例


以下是一个简单的Java程序，用于输出“Hello World!”，并对其代码进行详细分析：

        
package PgName;

import java.util.*; // 本例中只是提及，并未使用，可以删除不影响操作

public class HelloWorld {
    public static void main(String[] args) {
        System.out.print("Hello World!");
        // 在控制台打印“Hello World!”字符串
    }
}

代码分析

  • package PgName;
    包名，用于定位文件所在的目录结构。在Java中，包名通常与文件系统的目录结构相对应。例如，`package PgName;` 表示该文件位于 `PgName` 包中。
  • import java.util.*;
    导入包，用于调用某个包内的类和方法。`java.util.*` 表示导入 `java.util` 包中的所有类。虽然在这个简单的程序中没有使用到 `java.util` 包，但在更复杂的程序中，导入包是常见的操作。
  • public class HelloWorld {xxx}
    类声明，`public` 是访问修饰符，表示该类可以被其他类访问。`HelloWorld` 是类名，类名通常与文件名一致。
  • public static void main(String[] args) {xxx}
    程序主入口，`public` 表示该方法可以被外部调用，`static` 表示该方法是静态的（无需创建类的实例即可调用），`void` 表示该方法没有返回值。`main` 是程序的入口点，`String[] args` 是命令行参数数组。
  • System.out.print("Hello World!");
    输出语句，`System.out.print` 是Java的标准输出方法，用于在控制台打印内容。`"Hello World!"` 是输出的字符串。
  • {xxx}
    代码块，用大括号 `{}` 包裹的代码块表示方法或类的主体部分

常见错误

在刚开始学习Java时，初学者可能会犯以下错误：

  • 花括号 “ {} ” 未能闭合或正确闭合。
  • 将 `main` 错误地写成 `mian`。
  • 将 `void` 错误地写成 `viod`。
  • 忘记在类名后添加 `{}`。
  • 忘记在语句末尾添加分号 `;`。
  • else......

在学习过程中尽可能规范代码格式，以免产生更多不必要的麻烦以及阅读性很低的代码， 为后续自己或他人方便阅读，尽量遵循团队约定/大众使用的代码格式。

基本的Java命名规范（标识符）


在Java编程中，命名规范是代码可读性和可维护性的重要组成部分。 在命名时最好不要使用中文以及其他特殊字符， 要使用ASCII码表的字符。 最简单的，如'a-z', 'A-Z', '0-9', '_', '$'等等，且在命名时首字符不能是数字开头，且不能以关键字作为标识符。


类名（大驼峰命名）

类名通常使用大驼峰命名法（PascalCase），例如：

MyClass
HelloWorld

变量名（小驼峰命名）

变量名通常使用小驼峰命名法（camelCase），例如：

temp
control
label

常量名（大写+下划线）

常量名通常使用全大写，并用下划线分隔单词，例如：

IP_NUMBER
RULE

Java中的注释类型


Attend：注释和注解不是同一个概念，后续会提及。

以下是Java中常见的注释类型及其示例：


单行注释

使用“//”，例如：

// 此行字符将不会运行或被编译

多行注释

使用“/* */”，例如：

/* 此范围内任何字符将不会被运行或被编译 */

文档注释

使用“/** */”，例如：

/**
* 这些注释将会显示在一些地方
*/

文档注释（/** ... */）主要用于生成API文档。这些注释会被javadoc工具解析，并生成HTML格式的文档，供开发者查阅。

具体显示的 “地方” 是指由javadoc工具生成的HTML文档页面。这些页面通常包括：

  • 类的文档：描述类的功能、用途以及类的继承关系。
  • 方法的文档：描述方法的参数、返回值、异常以及方法的功能。
  • 字段的文档：描述类的成员变量及其用途。
  • 包的文档：描述包的功能和包含的类。

Java数据类型概述


Java中的数据类型分为基本数据类型和引用数据类型。


基本数据类型

基本数据类型是Java语言预先定义的类型，它们直接存储数据值，一个字节在二进制中是8bit，即'0000 0000'。以下是Java中的基本数据类型及其特点：

  • byte（占用1字节，范围：-128到127）
  • short（占用2字节，范围：-32768到32767）
  • int（占用4字节，范围：-231到231-1）
  • long（占用8字节，范围：-263到263-1）
  • float（单精度浮点数，占用4字节）
  • double（双精度浮点数，占用8字节）
  • char（字符类型，占用2字节，表示Unicode字符）
  • boolean（布尔类型，表示真或假，存储大小未明确规定，通常为1字节）

对于Java中，数据在从低精度到高精度时，会产生隐式转换，顺序为：

byte -> short -> char -> int -> long -> float -> double


若想进行强制类型转换，可在参数前加上 (数据类型)，比如你想将 char 显示为 byte，则有

            
System.out.println((byte)'C' + " " + (byte)'o' + " " + (byte)'d' + " " + (byte)'e');
// 输出结果将会是：67 111 100 101   （详情搜索ASCII码表）

值得注意的是，Java并不能把 boolean 转换成 short、int等类型，

而C++却可以转换：true -> 1；false -> 0。


引用数据类型

引用数据类型是通过引用访问的数据类型，它们存储对象的内存地址。以下是常见的引用数据类型：

  • 变量（存储引用，指向对象或数组）
  • 数组（存储多个相同类型的元素）
  • 类（自定义数据类型，封装数据和方法）
  • 接口（定义一组方法规范，由类实现）
  • 方法（定义类的行为，可以返回引用类型或基本类型）

特殊类型：String

String 是一种特殊的引用类型，它表示不可变的字符序列。

String 的特点：

  • 不可变：一旦创建，String 对象的内容不能被修改。
  • 引用类型：String 是一个类，它继承自 java.lang.Object。
  • 字符串常量池：字符串字面量会被存储在字符串常量池中，以节省内存。
  • 常用方法：如 length()、charAt()、substring() 等。

String在“改变”时的本质（无需求的话了解即可）


在Java中，String 是不可变对象。当对字符串进行“修改”操作时，实际上是在创建一个新的字符串对象。以下是一个示例： 

            
String st = new String("nihao") + new String("nijiaosm");

在这个例子中，st 是通过拼接两个字符串对象创建的。由于每次操作都会生成新的对象，因此它们的内存地址是不同的。

例如：

            
String str = "nihao";
String kbk = "ni";
String tp = "hao";
kbk = kbk + tp;  // 此时 kbk 是一个新的字符串对象

由于 kbk 是通过拼接生成的新对象，它的地址与 str 不同。因此，以下比较将返回 false：

            
boolean result = (str == kbk);  // 返回 false

然而，如果直接使用字符串字面量赋值：

            
String str = "nihao";
String kbk = "nihao";  // 或者 kbk = "ni" + "hao";

在这种情况下，str 和 kbk 都指向字符串常量池中的同一个对象，因此以下比较将返回 true： 

            
boolean result = (str == kbk);  // 返回 true

这是因为Java中的字符串常量池会自动重用相同的字符串对象，从而节省内存。

因此：

  • 字符串拼接会创建新的对象，因此地址不同。
  • 字符串常量池会重用相同的字符串对象，因此地址相同。

更多关于字符串常量池的内容，请参考Java官方文档或相关资料
